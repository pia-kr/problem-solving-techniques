# BOJ 2110 - 공유기 설치

## 문제 설명

도현이의 집 `N`개가 수직선 위에 존재한다.

각 집의 좌표는:

```text
x1, x2, x3 ... xN
```

이며, 같은 좌표를 가지는 집은 없다.

도현이는 집에 공유기 `C`개를 설치하려고 한다.

단:

- 한 집에는 공유기를 하나만 설치할 수 있다.
- 가장 인접한 두 공유기 사이의 거리를 가능한 크게 하고 싶다.

공유기를 적절히 설치했을 때,

```text
가장 가까운 두 공유기 사이 거리의 최댓값
```

을 구하는 프로그램을 작성하시오.

---

## 예시

집 좌표:

```text
1
2
4
8
9
```

공유기 개수:

```text
3
```

라고 하자.

공유기를:

```text
1, 4, 8
```

에 설치하면,

인접한 공유기 사이 거리는:

```text
3, 4
```

이고 최소 거리는:

```text
3
```

이다.

이 값이 가능한 최댓값이므로 정답은:

```text
3
```

이 된다.

---

## 입력

첫째 줄에:

```text
N C
```

가 주어진다.

- `N` : 집의 개수
- `C` : 공유기의 개수

둘째 줄부터 `N`개의 줄에는 집의 좌표 `xi`가 한 줄에 하나씩 주어진다.

---

## 제한

- `2 ≤ N ≤ 200,000`
- `2 ≤ C ≤ N`
- `0 ≤ xi ≤ 1,000,000,000`

---

## 출력

가장 인접한 두 공유기 사이 거리의 최댓값을 출력한다.

---

## 예제 입력

```text
5 3
1
2
8
4
9
```

---

## 예제 출력

```text
3
```

---

## 핵심 아이디어

집 좌표를 정렬한 뒤,

```text
공유기 사이 최소 거리
```

를 기준으로 이분 탐색(Binary Search)을 수행한다.

---

## 왜 이분 탐색인가?

예를 들어:

```text
최소 거리 = 3
```

으로 공유기 설치가 가능하다면,

```text
1, 2
```

같이 더 작은 거리도 항상 가능하다.

반대로:

```text
최소 거리 = 10
```

이 불가능하다면,

```text
11, 12 ...
```

같이 더 큰 거리도 불가능하다.

즉:

```text
가능 / 불가능
```

이 구간 형태로 나뉘므로 이분 탐색을 사용할 수 있다.

---

## 탐색 방식

현재 거리 `mid`를 기준으로:

```text
공유기 간 거리를 최소 mid 이상 유지하면서
C개 이상 설치 가능한가?
```

를 검사한다.

가능하면:

```text
거리를 더 늘려보기
```

불가능하면:

```text
거리를 줄이기
```

를 반복하여 최댓값을 찾는다.
